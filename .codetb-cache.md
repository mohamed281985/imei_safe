# IMEI Safe Haven - Comprehensive Project Analysis

## 1. PROJECT OVERVIEW

**Project Name:** IMEI Safe Haven
**Project Type:** Mobile-first Web Application (Android via Capacitor)
**Primary Language/Framework:** TypeScript, React 18, Vite, Capacitor

**Short Description:**
A comprehensive mobile platform for phone security that allows users to register phones by IMEI, report lost/stolen devices, search for phones, transfer ownership between sellers and buyers, and manage business listings. The application features a full Arabic/RTL interface with multilingual support (English, Arabic, French, Hindi), integrates with Paymob for payments, and uses Supabase for backend services with a dedicated Express.js server for payment processing and secure operations.

### Authentication Flow

```
1. User enters credentials
2. LoginForm.tsx → axiosInstance.post('/api/auth/login')
3. Server validates → creates Supabase session
4. Server generates CSRF token via doubleCsrfProtection
5. Frontend receives session → AuthContext updates
6. All subsequent requests include:
   - Authorization: Bearer <token>
   - X-CSRF-Token: <csrf-token>
```
